SharePoint Online – Full Implementation Guide

Export Guide
This guide tells you exactly where to go and what to paste in SharePoint Admin Center, SharePoint Online site, Power Automate, and Power Apps — step by step for each feature.

1. SharePoint Site Setup – Home Page

Where to go: Go to https://admin.microsoft.com → SharePoint Admin Center → Active Sites → Create → Communication Site
  1. Create a Communication Site named "Company DMS" at /sites/dms
  2. Set site language to English, time zone to your region
  3. Go to Site Settings → Site Information → Set site logo and title
  4. To show Recent Files: Add the "Highlighted Content" web part → Filter by: Recently Modified
  5. To add Search Bar: Add "Search Box" web part to the home page
  6. To add Departments grid: Add "Quick Links" web part → Select each library icon and link
  7. For Personalized views: Use "News" web part + "My Feed" web part
Make sure to enable Modern Experience in all document libraries (not Classic). Go to Library Settings → Advanced Settings → List experience: New experience.

2. Document Libraries Configuration

Where to go: Your SharePoint Site → + New → Document Library
  1. Create one Document Library per department: Engineering, Finance, Legal, Marketing, IT, HR, Procurement
  2. Create a master "Projects" library with sub-folders per project
  3. For each library: Go to Library Settings → Versioning Settings:
    • Enable: Require content approval → Yes
    • Enable: Create a version each time you edit a file → Major and minor versions
    • Keep drafts for: Only users who can approve
  4. Set Document Information Panel: Go to Content Types → Open in Word → Document Properties
Recommended: Use Content Types for each document type (Contract, Policy, Report, NDA) — Go to Site Settings → Site Content Types → New Content Type

3. Metadata Columns & Tag Management (Term Store)

Where to go (Term Store): SharePoint Admin Center → Content services → Term store
  1. In Term Store, create a Term Group named "DMS Tags"
  2. Inside it, create Term Sets: Document Types, Departments, Projects, Clients, Products, Status
  3. Add terms to each set (e.g., Contract, NDA, Policy, Invoice under "Document Types")
  4. To allow self-service tag creation: Open a Term Set → Set Submission Policy: Open
Where to go (Metadata Columns): Document Library → Library Settings → Create Column
  1. Create column: Tags → Type: Managed Metadata → Point to DMS Tags term set → Allow multiple values: Yes
  2. Create column: Project Name → Type: Single line of text
  3. Create column: Client → Type: Single line of text
  4. Create column: Product → Type: Single line of text
  5. Create column: Priority → Type: Choice → Values: 1, 2, 3
  6. Create column: Document Status → Type: Choice → Values: Draft, In Review, Approved, Released, Archived
  7. Create column: Version Label → Type: Single line of text
  8. Create column: Change Description → Type: Multiple lines of text
Folder Metadata Inheritance: This requires a Power Automate flow. When a file is added to a folder, the flow reads the folder's metadata columns and copies them to the new file. See Section 7.

4. Advanced Search Configuration

Where to go: SharePoint Admin Center → More features → Search → Manage Search Schema
  1. Find the Crawled Property for each metadata column you created (they appear as ows_Tags, ows_ProjectName, etc.)
  2. Map each crawled property to a Managed Property — create new ones named DmsTag, DmsProject, DmsClient, DmsProduct
  3. For each Managed Property, check: Searchable ✓, Queryable ✓, Retrievable ✓, Refinable – Yes - active
  4. After mapping: Go to Search → Manage Search Schema → Run a Full Crawl
  5. To add Search Refiners on search results page: Edit the Search Results page → Add Refinement web part → Add your managed properties
For the search page in your DMS site: Use a Search Center page or add a Search Results web part on a new page → Site Pages → New Page → Named "Advanced Search"

5. Check-In / Check-Out Setup

Where to go: Document Library → Library Settings → Versioning Settings
  1. Set Require Check Out: Yes → This forces users to check out before editing
  2. Users check out via: Select file → … (ellipsis) → More → Check Out
  3. Visual indicator: Checked-out files show a green arrow icon next to the file name in the library view
  4. To check in: Select file → … → Check In → Enter version comment
  5. Auto-release after 3 months: Create a Power Automate Flow:
    • Trigger: Recurrence → Daily
    • Action: Get Files (SharePoint) with filter: CheckedOutDate le utcNow(-90d)
    • Action: Check In File with comment: "Auto-released after 90 days"
    • Action: Send email to the user notifying of auto-release
// Power Automate Expression – Check if file is overdue checkout
addDays(item()?['CheckedOutDate'], 90)

6. Version Control Configuration

Where to go: Library Settings → Versioning Settings
  1. Enable Major and Minor Versions (Major = Published, Minor = Draft)
  2. To mark a Released Version: Select file → Version History → Promote to Major Version (Publish)
  3. To add mandatory Change Description:
    • Go to Library Settings → Versioning Settings → Require Check Out: Yes
    • When checking in, SharePoint will prompt for a version comment
    • To enforce non-empty comment: Create a Power Automate flow triggered on Item Created or Modified → Check if comment is blank → reject upload and notify user
  4. Version Label column: Add metadata column (single line) → Fill manually or auto-fill via Power Automate
  5. To view version history: Select file → Version History → Shows all versions with date, user, and comment

7. Folder Structure Automation

Where to go: Power Automate → Create → Automated Cloud Flow → Trigger: SharePoint – When an item is created (in a Project Template List)
  1. Create a SharePoint List called "Project Register" with columns: Project Name, Client, Board Name, Template Type
  2. In Power Automate, trigger: When a new item is created in "Project Register"
  3. Add action: Create a new folder in the target library → Path: /Projects/[ProjectName]
  4. Repeat for each sub-folder: 01_Base_Documents, 02_Contracts, 03_Work_Plans, 04_Correspondence, 05_Reports, 06_Approvals_Signatures
  5. To update folder name: Add action Update File with new name combining Board Name + Project Name
  6. Folder Metadata Inheritance Flow:
    • Trigger: When a file is created in the library
    • Get parent folder properties using Send HTTP Request to SharePoint → Read folder metadata
    • Update the new file's metadata columns with the folder's tag values using Update File Properties
// SharePoint REST API – Get folder metadata
GET _api/web/GetFolderByServerRelativeUrl('/sites/dms/docs/[FolderPath]')/ListItemAllFields
?$select=Tags,ProjectName,Client

8. Signature Workflow (Power Automate + API)

Where to go: Power Automate → Create → Automated Cloud Flow
  1. Create a SharePoint List called "Signature Requests" with columns: DocumentURL, Signers (multi-value), CurrentSignerIndex, Status, Priority, DueDate, CreatedBy
  2. Start Workflow Flow:
    • Trigger: When item is created in "Signature Requests"
    • Get the first signer from the Signers array
    • Send email to first signer with document link and sign button
    • Update CurrentSignerIndex = 0, Status = "In Progress"
  3. Process Signature Flow:
    • Trigger: When item is modified in "Signature Requests" (when signer acts)
    • If approved: Advance to next signer, send email to next signer
    • If last signer: Set Status = "Completed", notify creator
    • If rejected: Set Status = "Rejected", return to creator with comments
  4. PDF Conversion: Use Encodian or Adobe Sign connector in Power Automate to convert Word → PDF and embed signature
  5. Alternative (Adobe Acrobat Sign): Use the Adobe Sign Power Automate connector → Action: Send for Signature → it handles embedding and returning the signed PDF
  6. Restart Workflow: Add a button in Power Apps or a SharePoint list button that resets CurrentSignerIndex = 0 and Status = "Pending", then triggers the flow
For legally binding digital signatures with certificate embedding, integrate with DocuSign (connector available in Power Automate) or Adobe Sign. Both have native SharePoint connectors.

9. Email Notifications (Power Automate)

Where to go: Power Automate → All flows → Create each flow below
  1. Notify Next Approver: Triggered when Signature Request status changes → If new signer assigned → Send email with action buttons
  2. Notify Creator on Completion: Triggered when Status = "Completed" → Send email to creator with signed document link
  3. Daily Reminder:
    • Trigger: Recurrence → Every 1 day at 08:00
    • Action: Get all open Signature Requests where DueDate is within 3 days
    • Action: Send email to each signer with pending items
  4. Weekly Summary:
    • Trigger: Recurrence → Every Monday at 09:00
    • Action: Query all pending items per user
    • Action: Send HTML email with summary table
// Power Automate – Send formatted HTML email
Action: Send an email (V2) – Office 365 Outlook
To: @{variables('SignerEmail')}
Subject: [Action Required] Document Awaiting Your Signature
Body: Use "Send email with options" for Approve/Reject buttons

10. Permissions, Security & Active Directory

Where to go: SharePoint Site → Site Settings → Site Permissions → Create Groups
  1. Create SharePoint Permission Groups:
    • DMS-Readers → Read only
    • DMS-Contributors → Edit/Upload
    • DMS-Managers → Full control of their library
    • DMS-Admins → Site owner level
  2. Sync with Active Directory: Go to Azure Active Directory (Entra ID) → Groups → Create Security Groups matching department structure → Add users
  3. In SharePoint: Add each AAD Group to the appropriate SharePoint Permission Group
  4. Folder-level permissions: Select folder → Share → Manage Access → Stop Inheriting → Assign specific groups
  5. Manager identification: In Power Automate flows, use the Office 365 Users connector → Get Manager (V2) action to retrieve the manager of any user
  6. For AD hierarchy in reports: Use Graph API action in Power Automate → GET /users/{id}/manager
// Power Automate – Get user's manager from AAD
Action: Office 365 Users – Get Manager (V2)
User (UPN): @{triggerBody()?['Author']?['Email']}
// Returns: manager's name, email, department

11. Automated Reporting

Where to go: Power Automate → Scheduled Flows
  1. Monthly Checked-Out Report to Users:
    • Trigger: Recurrence → Day 1 of every month
    • Action: Get files from all libraries where CheckOutUser is not empty
    • Action: Group results by user
    • Action: For each user, send email with HTML table of their checked-out documents
  2. Monthly Manager Report:
    • Trigger: Recurrence → Day 1 of every month
    • Action: Get all users in each AD group
    • Action: For each user, get their manager using Office 365 Users connector
    • Action: Build HTML report per manager with team's document status
    • Action: Send email to each manager
  3. Power BI Integration (Optional): Connect Power BI to SharePoint lists → Build visual dashboards → Embed in SharePoint using Power BI web part
// Power Automate – Filter checked-out files (SharePoint REST)
GET _api/web/lists/getbytitle('Documents')/items
?$filter=CheckedOutUserId ne null
&$select=Title,CheckedOutDate,CheckedOutUserId,FileRef

12. Migration from VSS

Tool to use: SharePoint Migration Tool (SPMT) — Free Microsoft tool. Download from: aka.ms/SPMT
  1. Install SPMT on the migration source machine (where VSS files are accessible)
  2. Open SPMT → Sign in with your Microsoft 365 admin account
  3. Select File Share as source → Browse to the VSS export folder
  4. Select the destination: Your SharePoint site URL + target library
  5. In migration settings:
    • Enable: Preserve file timestamps
    • Enable: Only migrate the latest version
    • Enable: Migrate file metadata (where applicable)
  6. Run a Scan first to identify issues before migrating
  7. Run migration in batches (recommend 5,000 files per batch)
  8. After migration: Verify file count, run a full search crawl, check metadata mapping
File path length in SharePoint Online must be under 400 characters. Run SPMT scan to identify and rename long paths before migration.
For 40,000 files: Expect ~4-8 hours of migration time depending on file sizes and network speed. Run during off-peak hours.

13. Sharing & External Access

Where to go: SharePoint Admin Center → Policies → Sharing
  1. Set External Sharing level for the site: "Existing guests and new guests" or "Anyone with a link" depending on policy
  2. Set Default link expiration: 14 days → SharePoint Admin Center → Policies → Sharing → Set expiration days: 14
  3. To share a file: Select file → Share → Anyone with the link → Set expiration → Copy Link
  4. Track download confirmation: Enable SharePoint Access Requests + review audit logs. Go to Microsoft 365 Compliance Center → Audit → Search → Filter by "Downloaded file" activity
  5. External access without guest accounts: Use "Anyone with the link" sharing → Recipients don't need an account
  6. Open in Outlook Web panel: From any SharePoint file → Share → Outlook → Opens Outlook compose window with file link pre-filled
  7. To restrict sharing to only admins: Site Settings → Permissions → Site Sharing: "Only site owners can share"
